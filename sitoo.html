Scenario Rete Cisco negli Uffici

Analisi e progettazione della rete Cisco per una gestione ottimale degli uffici.

Descrizione dello Scenario

Questo scenario descrive una rete di uffici composta da quattro sedi collegate tramite tre router. Ho iniziato assegnando ad ogni rete un indirizzo di rete: Rete Gialla: 192.168.0.0 Rete Azzurra: 192.168.1.0 Rete Rosa: 192.168.2.0 Rete Rossa: 192.168.40.0 Successivamente ho attivato il DHCP su ogni server e ho assegnato ad ogni server l'indirizzo di rete da cui iniziare: Server 0: Default gateway 192.168.0.1 dns Server: 8.8.8.8 start ip address: 192.168.0.10 subnet mask: 255.255.255.0 Il server 0 assegna gli indirizzi anche alla sottorete Azzurra quindi Default Gateway: 192.168.1.1 dns server 8.8.8 start ip address: 192.168.1.10 subnet mask: 255.255.255.0 Ora passiamo al Server 2 attiviamo il DHCP e assegniamo gli indirizzi: Default Gateway: 102.168.40.1 dns server: 11.11.11.11 start ip address: 192.168.40.10 subnet mask: 255.255.255.0 Infine andiamo al server 3: Default Gateway: 192.168.2.1 dns server: 10.10.10.10 start ip address: 192.168.2.10 subnet mask: 255.255.255.0 Ora passiamo alla configurazione dei Router. Router 0: FastEthernet0/0 : 192.168.0.1 subnet mask: 255.255.255.0 FastEthernet1/0 : 192.168.1.1 subnet mask: 255.255.255.0 Serial2/0 : 200.100.50.1 subnet mask: 255.255.255.0 Serial3/0 : 200.50.50.1 subnet mask: 255.255.255.0 Router 1: FastEthernet0/0 : 192.168.2.1 subnet mask: 255.255.255.0 Serial2/0 : 200.50.50.2 subnet mask: 255.255.255.0 Serial3/0 : 200.150.50.1 subnet mask: 255.255.255.0 Router 2 : FastEthernet0/0 : 192.168.40.1 subnet mask: 255.255.255.0 Serial2/0 : 200.100.50.1 subnet mask: 255.255.255.0 Serial3/0 : 200.150.50.2 subnet mask: 255.255.255.0 Successivamente ho configurato la VLAN sulla sottorete Gialla: Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name Sottorete1 Switch(config-vlan)# exit Switch(config)# interface fastEthernet 0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# exit Switch(config)# interface fastEthernet 0/24 Switch(config-if)# switchport mode trunk Switch(config-if)# switchport trunk allowed vlan 10 Switch(config-if)# exit Infine ho fatto diverse simulazioni per provare tutto lo scenario e hanno avuto successo.
